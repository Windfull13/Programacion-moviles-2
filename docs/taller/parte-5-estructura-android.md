# Parte 5 — Estructura de un proyecto Android

## Objetivo
Identificar los archivos y carpetas que intervienen en una app Android con interfaz XML.

## Elementos importantes
- `AndroidManifest.xml`: declara la actividad principal y la configuración de la app
- `MainActivity.kt`: punto de entrada de la interfaz
- `res/layout/`: archivos XML de diseño
- `res/values/`: strings, colores y temas
- `build.gradle.kts`: dependencias y funciones del módulo

## En este proyecto
Los archivos relevantes son:
- `app/src/main/AndroidManifest.xml`
- `app/src/main/java/com/example/miprimeraapp/MainActivity.kt`
- `app/src/main/res/layout/activity_main.xml`
- `app/src/main/res/layout/item_tarea.xml`
- `app/build.gradle.kts`

## Idea clave
Android combina lógica en Kotlin con vistas declaradas en XML; el enlace entre ambos, en este taller, se hace con View Binding.
